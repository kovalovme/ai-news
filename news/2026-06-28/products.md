## Figma Config 2026: Code Layers, Figma Motion, and AI Generative Plugins <a id="figma-config-2026"></a>

**Source:** [Figma Blog](https://www.figma.com/blog/config-2026-recap/) · **Type:** launch · **Time (UTC):** Jun 24

Figma's annual Config conference (June 24) delivered its most developer-facing feature set to date. Four major additions:

**Code Layers** — Designers can convert any design layer into a live code layer with a single click, then clone a repo or extract existing component flows into the canvas. From there: iterate via AI chat, edit directly in Figma's code composer, or import npm packages (motion libraries, 3D frameworks, etc.) to add interactivity. Code layers round-trip back to editable design layers. Rollout begins July 2026.

**Figma Motion** — A timeline-based animation tool is now native to Figma Design. Users create keyframe animations, apply presets, and animate components that auto-sync across all screens. Export targets: CSS, JSON, React, MP4, WebM, Animated SVG, and GIF. Animations are MCP-compatible — a coded animation frame can be passed directly to a coding agent for implementation.

**AI Shader Generation** — A text-prompt or image-reference workflow generates custom GLSL shader fills and effects, adjustable and stackable with other Figma effects.

**Generative Plugins** — Non-engineers can describe a custom tool in natural language; Figma's AI builds it as a native-feeling canvas plugin. Plugins can be shared across teams or published to the community.

**Why it matters:** The code-layer round-trip collapses the design-to-dev handoff that has defined design tooling for a decade, and MCP-compatible animation export means Figma-authored motion can flow directly into coding agent contexts (Claude Code, Codex) without format translation.
