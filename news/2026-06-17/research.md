# Research — 2026-06-17

## "Running local models is good now" — Vicki Boykis <a id="local-models-good-now"></a>

**Source:** [Vicki Boykis](https://vickiboykis.com/2026/06/15/running-local-models-is-good-now/) · **Type:** technical essay · **Time (UTC):** Jun 15

Boykis, an ML engineer known for practitioner-focused writing, argues that locally-run language models have crossed a practical threshold for real engineering work. She reports using Gemma-4 for agentic coding tasks — code refactoring, test generation, and documentation — and estimating approximately 75% of the output quality of frontier API models. The post reached #1 on Hacker News (1,280 points) and generated substantial discussion on where the local/cloud tradeoff line now sits for specific workloads, with Blackwell-class consumer GPUs making sub-30B models increasingly viable on local hardware.

**Why it matters:** A shift from "local models are an interesting experiment" to "local models are good enough for real daily work" has direct consequences for cost control, data privacy, offline deployment, and the long-term leverage of frontier API providers — particularly as the Fable 5 export-control event has made API dependency newly visible as a risk.
